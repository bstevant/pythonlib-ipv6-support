\section{IPv6, pourquoi?}

\frame{
	\frametitle{Une tendance\ldots}
	Aujourd'hui tout converge vers Internet (\textit{internet centrism}) :
	\begin{itemize}
		\item Téléphonie
		\item \'Echanges bancaires et boursiers
		\item Données de santé
		\item Gestion de l'énergie
		\item Contrôle de chaines de production
		\item Etc.
	\end{itemize}
	La convergence et l'interopérabilité de ces applications est assurée par IP (entre autres).
}

\frame{
	\frametitle{Un constat\ldots}
	IPv4 est une expérience qui a mal tournée (dixit V.Cerf)
	\begin{itemize}
		\item Partitionnement des réseaux public et privés
		\item Fin du modèle bout-en-bout
		\item \'Epuisement des resources d'adresses IP disponibles
	\end{itemize}
}

\frame{
	\frametitle{La solution ?}
	IPv6 a tout pour plaire:
	\begin{itemize}
		\item Un espace d'adressage quasi-infini
		\item Un retour au modèle bout-en-bout
		\item L'auto-configuration des paramètres réseau
		\item et plein d'autres \textit{goodies} (Mobilité IP, segment routing, etc.)
	\end{itemize}
}

\frame{
	\frametitle{Transition à tous les étages}
	IPv6 n'est pas rétro-compatible avec IPv4 ! (\textit{it's a feature})
	\begin{itemize}
		\item IPv6 impose un nouveau plan d'adressage
		\item Le réseau sera majoritairement double-pile
		\item Les applications devront explicitement supporter IPv6
	\end{itemize}
